# Text-to-SQL PoC 产品需求文档 (PRD)

## 1. 产品定位

Matrix VC 风投机构内部数据分析工具。用户通过自然语言提问，系统自动转换为 SQL 查询并执行，以自然语言返回结果。本阶段为 PoC 验证，聚焦 Text-to-SQL 核心链路。

## 2. 目标用户

- VC 机构内部投资经理、分析师
- LP 关系管理人员
- 基金运营人员

## 3. 核心用户故事

| ID | 用户故事 | 验收标准 |
|----|---------|---------|
| US-1 | 作为投资经理，我想用自然语言查询基金数据，如"星辰基金上一季度总收入是多少" | 返回正确数值，以自然语言总结呈现 |
| US-2 | 作为分析师，我想对比企业财务表现，如"哪些企业本季度净利润增长率最高" | 返回排序后的企业列表及关键指标 |
| US-3 | 作为 LP 关系人员，我想查询 LP 出资信息，如"国盛保险的实缴总额是多少" | 返回精确金额 |
| US-4 | 作为用户，我想进行多轮对话追问，如先问"星辰基金的 AUM"，再追问"它的收益率呢" | 系统能理解上下文，正确回答追问 |
| US-5 | 作为用户，当我的问题与业务无关时，我希望得到明确的拒绝提示 | 返回友好拒绝消息，说明系统能力范围 |
| US-6 | 作为用户，当我的问题可能相关但缺少上下文时，系统应追问我补充信息 | 返回追问消息，提示用户补充具体查询对象或条件 |
| US-7 | 作为用户，当查询出错时，系统应自动修正并重试，而非直接报错 | SQL 错误后自动重试最多 3 次，给出最终结果或友好提示 |

## 4. 功能范围

### 4.1 包含（In Scope）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 自然语言查询 | 用户输入中文自然语言，系统生成 SQL 并返回自然语言结果 | P0 |
| 多轮对话 | 支持上下文追问，自动消解代词指代 | P0 |
| SQL 安全 | 仅允许 SELECT 查询，拒绝所有写操作 | P0 |
| 问题相关性过滤 | 通过 LLM 判断问题是否与 VC 业务相关，不相关则拒绝，缺少上下文则追问用户 | P1 |
| 错误自动重试 | SQL 执行失败时自动修正重试 | P1 |
| 表结构查看 | 用户可查看可用的表及字段说明 | P1 |
| 对话历史管理 | 支持 `/clear` 清空对话，重新开始 | P2 |
| Debug 模式 | CLI 支持 `--debug` 参数，实时展示 SQL 查询和工具调用过程 | P2 |

### 4.2 不包含（Out of Scope）

- 多系统联动 / RAG
- 细粒度权限控制（统一高权限）
- 数据写入 / 修改操作
- 实时数据流接入
- 多租户 / 多数据库支持

## 5. 业务数据域

系统覆盖以下 7 张业务表：

| 表名 | 业务含义 | 关键字段 |
|------|---------|---------|
| funds | 基金基本信息 | 名称、类型(VC/PE/FOF)、规模、状态 |
| companies | 被投企业 | 名称、行业、地区、投资阶段、状态 |
| fund_investments | 投资记录 | 基金→企业、金额、估值、持股比例、轮次 |
| shareholders | LP/股东 | 名称、类型、认缴总额 |
| fund_shareholders | 基金-LP 关系 | 认缴金额、实缴金额、占比 |
| quarterly_reports | 季度报告 | AUM、净值、收益率、累计投资额 |
| financial_statements | 企业财务 | 营业收入、净利润、总资产、总负债、员工数 |

测试数据规模：3 只基金、5 家企业、4 个 LP、20+ 投资记录、12 条季报、30+ 财务数据。

## 6. 交互方式

### 6.1 CLI 客户端

#### 输入

- 自然语言文本（中文）
- CLI 命令：`/quit`、`/clear`、`/tables`、`/schema`、`/help`

#### 输出

- 自然语言回答（中文），包含关键数据总结
- 金额单位统一为"亿元"，百分比保留 2 位小数
- 查询无结果时提示"未找到匹配数据"

### 6.2 Web 应用

#### 登录

- 用户名 + 密码登录，Token 认证（Bearer Token）
- 401 响应自动登出

#### 输入

- 自然语言文本（中文），Enter 发送
- 侧边栏快捷提问（预设示例问题）
- 侧边栏快捷指令：查看所有表、查看表结构、清空会话
- `/tables`、`/schema` 命令

#### 输出

- SSE 流式实时响应，逐步展示 AI 思考过程和最终结果
- Debug 模式下可展开查看 AI 思考、生成的 SQL、原始查询结果
- 自然语言回答（中文），包含关键数据总结
- 金额单位统一为"亿元"，百分比保留 2 位小数

#### 会话管理

- 支持新建、切换、删除对话
- 对话历史持久化到后端
- 首条消息自动生成对话标题

## 7. 非功能需求

| 类别 | 需求 |
|------|------|
| 安全性 | 仅允许 SELECT 查询，正则校验禁止 INSERT/UPDATE/DELETE/DROP |
| 可靠性 | SQL 错误自动重试（最多 3 次）；LLM API 错误不重试，直接报错 |
| 可用性 | CLI（rich 彩色输出）和 Web 界面两种交互方式 |
| 部署 | 本地运行，依赖内部 vLLM 推理服务 |
| 数据 | SQLite 文件存储，支持重建测试数据 |
| 会话持久化 | 对话历史持久化到 SQLite（非内存），服务重启不丢失 |

## 8. 错误处理

| 场景 | 用户感知 |
|------|---------|
| 问题与 VC 业务无关 | "抱歉，我是 VC 机构内部数据分析助手，只支持基金、投资、企业财务等业务相关的数据查询。" |
| 问题可能相关但缺少上下文 | 系统追问用户补充信息，如"请指明您想查询哪个基金的收益率" |
| 问题含糊（多轮对话中） | 系统通过 LLM 结合对话历史自动理解，直接得到正确回答 |
| SQL 执行出错 | 系统自动重试；重试耗尽后提示"查询多次失败，请尝试换一种问法" |
| 查询无结果 | "未找到匹配数据" |
| 非 SELECT 操作 | 拒绝执行，返回错误信息 |
| LLM 服务不可用 | 报错，提示检查网络和服务配置 |

## 9. 技术选型

| 组件 | 选型 |
|------|------|
| 运行时 | Python 3.12+，uv 包管理 |
| AI 框架 | LangChain v1（`create_agent` + middleware） |
| LLM | Qwen3.5-27B（vLLM 部署，OpenAI 兼容 API） |
| API 框架 | FastAPI + uvicorn（后端 REST API 服务） |
| 客户端通信 | httpx（CLI 客户端 HTTP 调用） |
| 会话持久化 | langgraph-checkpoint-sqlite（`SqliteSaver`） |
| 数据库 | SQLite（业务数据 `vc_test.db` + 会话 `sessions.db`） |
| Web 前端 | React 19 + TypeScript + Vite 8 + Tailwind CSS 4 |
| Web 通信 | SSE（Server-Sent Events）流式响应 |
| Web 认证 | Bearer Token（localStorage 存储） |
| CLI | rich |
| 测试 | pytest |

## 11. 启动方式

```bash
# 1. 初始化测试数据库（首次运行或需要重建时）
cd backend && uv run python data/init_db.py

# 2. 启动后端 API 服务
cd backend && uv run text-to-sql-server

# 3. 启动 CLI 客户端（另一个终端）
cd cli-client && uv run text-to-sql-cli          # 正常模式
cd cli-client && uv run text-to-sql-cli --debug  # Debug 模式

# 4. 启动 Web 应用（另一个终端）
cd web-app && npm install   # 首次安装依赖
cd web-app && npm run dev   # 开发模式（热更新）
```

## 12. 里程碑

| 阶段 | 目标 | 交付物 |
|------|------|--------|
| M1: 基础设施 | 项目骨架、数据库、配置管理 | 可运行的空项目 + 测试数据库 |
| M2: 核心 Agent | Text-to-SQL 主链路打通 | 能回答单轮自然语言查询 |
| M3: 质量保障 | 输入检查、错误重试、多轮对话 | 稳定可靠的用户体验 |
| M4: 前后端分离 | 拆分 backend + cli-client，HTTP API 通信 | 独立部署的后端和 CLI 客户端 |
| M5: Web 应用 | React SPA 前端，SSE 流式响应，Token 认证 | Web 界面 + 会话管理 |
| M6: 测试验收 | 单元测试覆盖、端到端验证 | 可交付的 PoC |

## 13. 验收标准

### 基本查询能力

- [ ] "星辰成长基金的规模是多少" → 返回正确金额
- [ ] "哪些企业在人工智能行业" → 返回匹配企业列表
- [ ] "恒远 PE 基金投资了哪些企业" → 返回被投企业列表及金额

### 聚合与排序

- [ ] "各基金的 AUM 排名" → 按降序返回
- [ ] "2024 年所有企业的总营收" → 返回汇总数值

### 多轮对话

- [ ] 先问"星辰基金的 AUM"，再问"它的收益率呢" → 第二问正确引用星辰基金
- [ ] `/clear` 后提问不引用之前对话上下文

### 边界情况

- [ ] "今天天气怎么样" → 被拦截，返回拒绝提示
- [ ] "它的收益率呢"（首轮，无上下文） → 追问用户指明具体基金
- [ ] 先问"星辰基金上半年的季报"，再问"那么下半年呢？" → 正确理解上下文，返回下半年季报
- [ ] 故意输入 `DELETE FROM funds` → 被拒绝执行
- [ ] `/tables` → 显示所有表名及说明
- [ ] `/schema` → 显示完整 DDL
- [ ] `/help` → 显示帮助信息

### Debug 模式

- [ ] `--debug` 模式下能看到生成的 SQL 查询语句
- [ ] `--debug` 模式下能看到工具调用和工具返回结果

### Web 应用

- [ ] 登录页面输入用户名密码后可成功登录
- [ ] 登录后进入聊天界面，侧边栏显示会话列表
- [ ] 新建对话后发送自然语言问题，SSE 流式返回结果
- [ ] 多轮对话中追问能正确引用上下文
- [ ] 侧边栏快捷提问点击后自动发送对应问题
- [ ] 快捷指令（/tables、/schema）正常工作
- [ ] Debug 模式开关打开后，消息下方可展开查看 AI 思考、SQL、原始结果
- [ ] 切换对话后消息正确加载
- [ ] 删除对话后从列表消失
- [ ] 401 响应自动登出回到登录页
